## 앱 생성

### 1단계: Google Play Console 접속
- URL: https://play.google.com/console
- Google 계정으로 로그인

### 2단계: 새 앱 생성
1. **"앱 만들기"** 클릭
2. **앱 이름**: `OilNow+`
3. **기본 카테고리**: `지도 및 내비게이션` 또는 `라이프스타일`
4. **앱 유형**: `Android 앱`
5. **기본 스토어 목록 언어**: `한국어`
6. **"만들기"** 클릭

### 3단계: 앱 정보 입력
- **패키지 이름**: `com.oilnowplus.app`
- **앱 카테고리**: 지도 및 내비게이션
- **콘텐츠 등급**: 일반 (G)

## 내부 테스트 설정

### 1단계: 테스트 트랙 생성
1. 왼쪽 메뉴에서 **"테스트 → 내부 테스트"** 클릭
2. **"새 릴리스 만들기"** 클릭

### 2단계: APK 업로드
1. **"APK 업로드"** 클릭
2. 빌드된 `app.apk` 파일 선택
3. **"검토"** 클릭

### 3단계: 릴리스 정보 입력
- **릴리스 이름**: `v1.0.0 - 내부 테스트`
- **릴리스 노트**: 
  ```
  OilNow+ v1.0.0
  - 초기 버전 출시
  - 주유소 정보 조회
  - 실시간 유가 정보
  - 지도 기반 검색
  ```
- **"저장"** 클릭
- **"내부 테스트로 롤아웃"** 클릭

---

## 테스터 초대

### 1단계: 테스터 그룹 생성
1. **"테스트 → 내부 테스트"** 에서 **"테스터 관리"** 클릭
2. **"테스터 그룹 추가"** 클릭
3. **그룹 이름**: `OilNow+ 베타 테스터`
4. **"만들기"** 클릭

### 2단계: 테스터 추가
1. **"테스터 추가"** 클릭
2. 테스터 이메일 주소 입력 (한 줄에 하나씩)
3. **"저장"** 클릭

### 3단계: 테스트 링크 공유
1. **"테스터 초대 링크"** 복사
2. 테스터들에게 이메일로 링크 전송
3. 테스터들이 링크를 클릭하면 자동으로 앱 설치 가능

## APK 업로드

### 자동 업로드 (GitHub Actions)

#### 1단계: Google Play Service Account 생성
1. Google Cloud Console: https://console.cloud.google.com
2. **"프로젝트 선택"** → **"새 프로젝트"** 클릭
3. **프로젝트 이름**: `OilNow+ Play Store`
4. **"만들기"** 클릭

#### 2단계: Service Account 키 생성
1. **"APIs 및 서비스"** → **"서비스 계정"** 클릭
2. **"서비스 계정 만들기"** 클릭
3. **서비스 계정 이름**: `oilnow-play-store`
4. **"만들기"** 클릭
5. **"키"** 탭 → **"새 키"** → **"JSON"** 선택
6. JSON 파일 다운로드

#### 3단계: GitHub Secrets 설정
1. GitHub 저장소 → **Settings** → **Secrets and variables** → **Actions**
2. **"New repository secret"** 클릭
3. **Name**: `GOOGLE_PLAY_SERVICE_ACCOUNT`
4. **Value**: 다운로드한 JSON 파일 내용 (전체 텍스트)
5. **"Add secret"** 클릭

#### 4단계: Google Play Console에서 권한 부여
1. Google Play Console → **"설정"** → **"API 액세스"**
2. **"서비스 계정 연결"** 클릭
3. 위에서 생성한 서비스 계정 선택
4. **역할**: `관리자` 선택
5. **"초대"** 클릭

### 수동 업로드

#### 로컬에서 APK 빌드
```bash
# 1. 의존성 설치
npm ci

# 2. APK 빌드
bash scripts/build-apk.sh

# 3. 결과 확인
ls -lh app.apk
```

#### Google Play Console에서 업로드
1. **"테스트 → 내부 테스트"** 클릭
2. **"새 릴리스 만들기"** 클릭
3. **"APK 업로드"** 클릭
4. 빌드된 `app.apk` 파일 선택
5. **"검토"** → **"내부 테스트로 롤아웃"** 클릭

## 📱 테스터 설치 방법

### 테스터가 해야 할 일

1. **테스트 링크 클릭**
   - 이메일에서 받은 링크 클릭

2. **Google Play Store에서 설치**
   - "설치" 버튼 클릭
   - 앱 설치 완료

3. **피드백 제공**
   - Google Play Store 앱 페이지에서 평가 및 리뷰 작성

---

## 🔄 업데이트 프로세스

### 새 버전 배포
1. 코드 수정 및 테스트
2. `package.json`에서 버전 업데이트
3. GitHub에 푸시 (자동 빌드 시작)
4. APK 빌드 완료 후 자동 업로드
5. Google Play Console에서 확인

---

## ⚠️ 주의사항

- **패키지 이름**: 반드시 `com.oilnowplus.app`로 설정
- **서명**: 릴리스 APK는 반드시 서명되어야 함
- **테스터 수**: 최대 100명까지 내부 테스트 가능
- **테스트 기간**: 제한 없음 (원하는 만큼 진행 가능)

## 📞 문제 해결

### APK 업로드 실패
- 패키지 이름 확인
- APK 서명 확인
- 버전 코드 확인 (이전 버전보다 높아야 함)

### 테스터가 앱을 설치할 수 없음
- 테스터 이메일 주소 확인
- 테스트 링크 유효성 확인
- Google 계정으로 로그인되어 있는지 확인

### 자동 배포 실패
- GitHub Secrets 설정 확인
- Service Account 권한 확인
- Google Play API 활성화 확인
